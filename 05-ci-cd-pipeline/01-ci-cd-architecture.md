# CI/CD 架构设计

## 1. 背景

在传统部署模式下：
- 手动部署效率低
- 容易出错
- 无法快速迭代

因此需要构建 CI/CD 系统，实现自动化交付。

---

## 2. 总体架构

CI/CD 流程如下：

Git → CI（构建）→ 镜像仓库 → CD（部署）→ Kubernetes

---

## 3. 组件说明

| 组件 | 作用 |
|------|------|
| Git | 代码管理 |
| Jenkins | 构建与流水线 |
| Docker | 镜像构建 |
| Registry | 镜像存储 |
| Kubernetes | 运行环境 |

---

## 4. 数据流

1. 开发提交代码
2. Jenkins 拉取代码
3. 构建镜像
4. 推送仓库
5. 更新部署

---

## 5. 架构特点

- 自动化
- 可重复
- 可追溯
- 可扩展
